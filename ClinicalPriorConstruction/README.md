# Clinical Prior Construction

This folder contains scripts for constructing clinical prior features from registered CT scans, segmentation masks, and patient-specific vascular-territory atlases.

## Contents

- `main.py`: extracts patient-level hemorrhage features into a CSV file.
- `calculate_region_attention_weights.py`: estimates regional and global prior parameters from the feature CSV.

## Requirements

- Python 3.10 or newer.
- NumPy.
- Pandas.
- SimpleITK.
- tqdm.
- scikit-learn.

## Expected Upstream Output

`main.py` expects patient-specific atlas files generated by `RegistrationAndSkullStripping/main.py`.

For each patient ID, the registration output folder should contain:

```text
result_v3/
  <patient_id>/
    individualized_annotation_in_preop_mni_affine.nii.gz
    individualized_annotation_in_postop_mni_affine.nii.gz
```

These individualized atlas files replace the need to read a single global `mni_vascular_territories.nii.gz` template.

## Input Layout

The CT and segmentation input folders should contain paired preoperative and postoperative NIfTI files in a flat structure:

```text
registered_ct_and_segmentation_folder/
  P001.nii.gz
  P001_seg.nii.gz
  P001-1.nii.gz
  P001-1_seg.nii.gz
```

The `-1` suffix marks the postoperative scan.

Segmentation labels are expected to be:

- `1`: IVH.
- `2`: SAH.
- `3`: Ventricle.

## Feature Extraction

Edit the configuration at the end of `main.py`:

```python
CONFIG = {
    "registration_result_root": "/path/to/RegistrationAndSkullStripping/result_v3",
    "data_roots": [
        "/path/to/registered_ct_and_segmentation_folder",
    ],
    "output_dir": "./outputs",
    "output_csv_name": "features.csv",
    "resume": True,
    "max_patients": None,
}
```

Then run:

```bash
python main.py
```

The output is a feature CSV, for example:

```text
outputs/features.csv
```

## Prior Parameter Estimation

`calculate_region_attention_weights.py` reads one or more feature CSV files. By default it reads:

```text
outputs/features.csv
```

The feature CSV must include an `mRS` column. Poor outcome is defined as:

```python
mRS >= 3
```

Run:

```bash
python calculate_region_attention_weights.py
```

The script writes:

```text
outputs/prior_params/
  region_metrics_10_pre.csv
  region_metrics_10_post.csv
  region_metrics_6_full_pre.csv
  region_metrics_6_full_post.csv
  region_prior_params_pre.csv
  region_prior_params_post.csv
  global_prior_params.csv
```

## Data Policy

No patient imaging data or clinical CSV files are included in this upload folder. Provide those files locally and update the configuration paths before running.
